# Онбординг и агентурный слой

## Проблема

В приложении много специальных терминов: raw, signature, FIFO, PnL, Jito, priority fee, trigger, coverage, SoT, verdict, cluster_context, repeat_wave, price_action, cross_chain_regime, Study, Analyze, Learn, Predict, Notify, Gather.

Если пользователь видит график, но не понимает термин, график превращается в шум. Если пользователь видит PASS и воспринимает это как BUY, это уже риск. Если агент говорит о вероятности, но не показывает риск и цену ошибки, это ложная уверенность.

## Решение

Основной пользовательский слой приложения должен быть на русском языке.

Онбординг встроен внутрь Iskra Forge как вкладка `Гайд / Агент`, а не как новая верхнеуровневая Streamlit page.

Агентурный слой должен быть режимным: пользователь видит, в каком режиме работает агент, какой результат ожидать и какие guardrails включены.

## UI

### Мини-подсказки

Термины получают маленький полупрозрачный значок `?`.
При клике раскрывается карточка:
- название термина;
- простое объяснение;
- зачем термин нужен в membot.

Подсказка должна быть читаемой: светлый заголовок, контрастный body text, тёплая строка `why`, высокий z-index.

### Вкладки

- `Кузница сигналов` — главный обзор.
- `Гайд / Агент` — FAQ, словарь, режимы агента и agent chat.
- `Raw` — сырые swaps/transaction-derived rows.
- `FIFO` — paired trades и учёт входов/выходов.
- `PnL по дням` — календарь прибыли/убытка.
- `Fee/Jito` — priority fee, ComputeBudget, Jito evidence.
- `Триггеры` — pre-buy и entry/exit hypotheses.
- `Отчёты` — markdown reports.
- `Загрузка` — upload артефактов.

## Термины, которые должны быть в словаре

- `cluster_context` — окружение токена, соседние сделки, похожие кошельки, режим рынка.
- `repeat_wave` — повторная волна похожей активности.
- `price_action` — поведение цены до и после входа.
- `cross_chain_regime` — режим, где импульс может прийти из другой сети или рынка.
- `Entry/exit hypotheses` — проверяемые предположения входа и выхода.
- `Study` — изучение доступных данных и missing artifacts.
- `Analyze` — анализ паттернов, разрывов, fee/Jito и drift.
- `Learn` — сохранение не-секретных уроков run/QA.
- `Predict` — оценка вероятности поддержки гипотезы.
- `Notify` — безопасные уведомления о состоянии и риске.
- `Gather` — план сбора недостающих данных из разрешённых источников.

## Агентурный слой v3

Агент изучает, анализирует, учится на прогонах, оценивает вероятность поддержки гипотезы, уведомляет о рисках, собирает информацию и предлагает следующий research-step.

Он не выдаёт финансовую рекомендацию, не отдаёт BUY/SELL и всегда показывает риск и цену ошибки.

### Режимы агента

#### Study

Назначение: понять, какие данные уже есть, чего не хватает и где начинается UNKNOWN.

Выход:
- snapshot источников;
- missing critical artifacts;
- свежесть и готовность к анализу;
- ограничение уверенности.

Guard: не делать выводы без raw/FIFO/fee якорей.

#### Analyze

Назначение: искать паттерны, несостыковки, fee/Jito следы, режимы поведения и слабые места гипотез.

Выход:
- evidence flags;
- drift/routing/fee/slippage risks;
- направления проверки;
- список ложных корреляций, которые нужно исключить.

Guard: не превращать корреляцию в причину и не выдавать dashboard claims как raw truth.

#### Learn

Назначение: фиксировать не-секретные уроки прогонов, QA-вердиктов и ошибок pipeline.

Выход:
- run notes;
- lessons;
- QA-memory;
- повторяемые ошибки.

Guard: не хранить secrets и не считать memory источником истины.

#### Predict

Назначение: оценивать вероятность поддержки гипотезы и вероятность прохождения следующей валидации.

Выход:
- UNKNOWN / LOW / MEDIUM / HIGH;
- evidence-score;
- риск;
- цена ошибки;
- следующий шаг проверки.

Guard: не обещать прибыль и не выдавать рекомендацию.

#### Notify

Назначение: формировать безопасные условия уведомлений по изменению данных, риска или качества покрытия.

Выход:
- watch conditions;
- риск-события;
- состояние свежести данных;
- degraded RPC/data alerts.

Guard: уведомления не должны звучать как BUY/SELL alerts.

#### Gather

Назначение: определить, какие данные надо подтянуть дальше из разрешённых источников.

Выход:
- data wishlist;
- source plan;
- freshness requirements;
- next query/run plan.

Guard: не собирать secrets и не использовать неразрешённые источники.

### Read-only tools

- `summarize_artifacts` — сводка доступных артефактов и строк.
- `explain_term` — объяснение термина из словаря.
- `inspect_fee_jito` — Fee/Jito/ComputeBudget разбор.
- `inspect_green_days` — проверка готовности green-days claim.
- `estimate_hypothesis_support` — вероятность поддержки гипотезы без BUY/SELL.
- `risk_price_explainer` — риск и цена ошибки.
- `propose_next_run` — следующий безопасный run-step.
- `study_mode` — изучить состояние данных.
- `analyze_mode` — разобрать паттерны и разрывы.
- `learn_mode` — сохранить не-секретный урок сессии.
- `predict_mode` — оценить вероятность поддержки гипотезы.
- `notify_mode` — сформировать условия безопасных уведомлений.
- `gather_mode` — построить план сбора недостающих данных.

## Boundary formula

```text
Агент оценивает вероятность и риск.
Агент не отдаёт приказ.
Raw artifacts остаются выше agent output.
```

## QA

PASS если:
- интерфейсные вкладки на русском;
- hero на русском;
- `?` подсказки читаемые;
- `cluster_context`, `repeat_wave`, `price_action`, `cross_chain_regime`, `Entry/exit hypotheses` есть в словаре;
- `Гайд / Агент` открывается;
- режимы Study/Analyze/Learn/Predict/Notify/Gather видны как карточки;
- radio selector режимов работает;
- быстрые кнопки режимов добавляют ответ агента;
- FAQ раскрывается;
- glossary search фильтрует термины;
- агент отвечает на:
  - `Study`
  - `Analyze`
  - `Learn`
  - `Predict`
  - `Notify`
  - `Gather`
  - `что дальше?`
  - `объясни FIFO`
  - `проверь fee/Jito`
  - `почему daily PnL UNKNOWN?`
  - `какой риск у trigger?`
  - `какие инструменты есть?`
- агент не выдаёт BUY/SELL;
- sidebar не получает новых top-level pages.
